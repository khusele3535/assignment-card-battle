## 🖼️ Даалгавар 1: Wireframes (10 оноо)

### Шаардлага

Дор хаяж **5 screen**-ийн wireframe зурах. Цаасан дээр гараар, эсвэл дижитал (Figma, Excalidraw, Sketch, Miro, Paper app) — аль ч арга болно.

### Заавал багтах screen-үүд

| # | Screen | Юу харагдах вэ? |
|---|--------|-----------------|
| 1 | **Main Menu** | New Game, Load Game, Options, Quit |
| 2 | **Card in Hand** | Нэг картны layout — нэр, мана, damage, rarity, description |
| 3 | **Battle Screen** | 2 player HP/mana, гарт буй карт, battlefield, ээлжийн тоо |
| 4 | **Card Played Animation** | Карт тоглогдох үеийн "өмнө"/"дараа" кадр |
| 5 | **Game Over** | Winner banner, тоглоомын статистик (тоглосон карт, damage dealt) |

### Үнэлгээ

- 5/10 — 5 screen байгаа

## 👥 Даалгавар 2: Персона (5 оноо)

### Шаардлага

**2 персона** үүсгэх — таны тоглоомыг хэн тоглох вэ? Эдгээр нь **бодит биш, гэхдээ итгэл төрүүлэхүйц** дүр. Тэднийг бодитой болгох нь тань таны design decision-ыг удирдана.

### Template (`docs/personas/README.md`-т хуулна уу)

```markdown
## Персона 1: [Нэр]

![Зураг / emoji]  📸 (optional)

| Шинж | Утга |
|------|------|
| Нэр | Түмэнжаргал |
| Нас | 21 |
| Мэргэжил | IT оюутан |
| Тоглоомын туршлага | Hearthstone 2 жил, Genshin Impact |
| Төхөөрөмж | Laptop + Terminal |

**Motivation (юу хүсэж байна вэ?):**
- Хичээл завсар 10-15 мин богино тоглоом
- Шинэ тактик бодож олох боломж

**Frustration (юу бухимдуулна вэ?):**
- Нарийн control-тай (mouse + keyboard аль аль шаардлагатай) тоглоом
- Маш удаан tutorial

**Таны тоглоом түүнд яагаад таалагдах вэ?**
- Бүгд keyboard-оор, 5 мин дотор үндсэн механик ойлгомжтой
- Долоон карт, stratege шаардсан сонголт

**Таны тоглоом түүнд яагаад бухимдуулж магадгүй вэ?**
- ASCII-ийн хязгаарлалт — анимаци бага
- Save/load байхгүй бол богино suspend боломжгүй

---

## Персона 2: [Өөр хэн нэгэн — өөр контекст]

(Өөр нас, өөр context, өөр motivation. Жишээ: "46 настай эх, багтай үед хүүтэйгээ тоглох үе шууд хэрэгтэй", эсвэл "13 настай сургуулийн сурагч, programming сонирхолтой".)
```

### Яагаад 2 персона?

Нэг биш олон хүн ашиглана. Хоёр өөр тохиолдолд "энэ карт хангалттай ойлгомжтой юу?" гэдэг нь **өөр хариулт** гарна. Таны дизайн аль нэгэнд нь сайхан болгохоор биш, хоёуланд нь ойлгомжтой байх ёстой.

## ✍️ Даалгавар 3: UX Writing Audit (5 оноо)

### Шаардлага

Таны тоглоомд байгаа **бүх string-ийг** (карт нэр, тайлбар, error message, menu label, instruction, game over text) **жагсааж**, дараах 4 шалгуураар үнэлэх:

1. **Clarity** — Programming сурагдаагүй хүн ойлгох уу?
2. **Tone** — Dramatic / friendly / minimal — ямар тонар сонгосон, бүх string энэ тонарт тохирч байна уу?
3. **Consistency** — "карт" / "card" / "тамга" — ижил зүйлд ижил үг хэрэглэсэн үү?
4. **Language policy** — 100% монгол уу, эсвэл карт нэр англи хэвээр үлдэх үү?

### Формат (`docs/ux-audit/README.md`-т)

```markdown
# UX Writing Audit

**Tone сонгосон:** 🗡️ Dramatic (RPG feel)
**Хэлний журам:** Монгол — гэхдээ карт нэр англиар үлдэнэ (брэнд recognition-д)

## String жагсаалт

| # | Анх бичигдсэн | Газар (файл) | Асуудал | Зөв болгосон |
|---|---------------|--------------|---------|-------------|
| 1 | "Error: Invalid input" | InputReader.java:45 | Dramatic биш | "Буруу оролт — дахин оролдоорой, дайчин!" |
| 2 | "HP: 18/30" | ConsoleRenderer.java:80 | OK | (үлдээсэн) |
| 3 | "You won!" | Game.java:120 | English + flat | "Ялалт чиний гарт! ⚔️" |
| 4 | "Fireball" | cards/AttackCard.java | Консистент биш — өөр картнуудын нэр монголоор | "🔥 Blazing Fireball" — бүгд картнууд ижилхэн англи-монгол холимог болгов |
| ... | ... | ... | ... | ... |

## Шийдсэн журам

- **Тонар:** Dramatic RPG — "дайчин", "зүрхэн дээр тусч" гэх мэт үг
- **Карт нэр:** Англи хэвээр + emoji ("🔥 Fireball"), брэнд recognition-д
- **Систем мессеж:** Монгол
- **Error:** Хэзээ ч "Error XXX" гэхгүй — "Юу болсон + дараа яах вэ" форматаар

## 3 гол сургамж

1. [Ямар зүйл бат гэж ойлгосон вэ?]
2. [Ямар 2 string-ийг хамгийн их сайжруулсан вэ?]
3. [Дараагийн тоглоомонд яаж хандах вэ?]
```

### Үнэлгээ

- 3/5 — Жагсаалт + зарим өөрчлөлт
- 4/5 — Tone тогтсон, бүх string түүнд тохирсон
- 5/5 — Consistency, language policy, journal хэлбэртэй ("3 сургамж") бүрэн

## 🧪 Даалгавар 4: User Test Report (5 оноо)

### Шаардлага

Таны тоглоомыг **3 бодит хүнээр** тоглуулах. Эдгээр хүмүүсийг урьдчилан **сурадаагүй** байх ёстой — Java мэддэг нөхрөөр туршуулахгүй.

### Даалгавар тэдэнд:

> "Энэ тоглоомыг нээгээд **нэг тоглоомыг амжилттай ялах** гэж оролд. Би хажууд нь чимээгүй зогсож байх болно."

### Ажиглах зүйлс:

- Хаана зогссон бэ? (confusion moment)
- Ямар үед "би юу хийх ёстой вэ?" гэж асуусан?
- Аль карт ойлгомжгүй санагдсан?
- Аль error message нь тэднийг зөв зам руу чиглүүлсэн / чиглүүлээгүй?

### Формат (`docs/user-test-report/README.md`-т)

```markdown
# User Test Report

## Хэн тестлэсэн бэ?

| # | Нэр | Нас | Тоглоомын туршлага |
|---|-----|-----|-------------------|
| 1 | Ирмүүн | 22 | TCG тогтмол тоглодог |
| 2 | Эгч | 35 | Mobile casual game |
| 3 | Ах | 17 | Console FPS тогтмол |

## Findings

### 🔴 Блокер (амжилтгүй болгосон зүйл)
1. **Ээлж дуусгах арга ойлгогдсонгүй** — 3/3 хүн "-1 оруулах" гэдгийг tutorial-гүйгээр ойлгоогүй
2. **BuffCard үр дүнг олж хараагүй** — 2/3 хүн Buff тоглосны дараа юу өөрчлөгдсөнийг мэдрээгүй

### 🟡 Жижиг бэрхшээл
1. Мана хүрэлцэхгүй үед error message нэг секундийн дараа алга болчихно — уншиж амжихгүй
2. HP bar-ын нарийн хэмжээ тодорхойгүй

### 🟢 Сайн талууд
1. Картны нэр + emoji тааламжтай байсан — 3/3 хүн "энэ нэр гоё" гэсэн
2. AI opponent нь "санаанд багтахгүй зүйл хийлгүй" гэж тайвшруулсан

## Хийсэн 3 сайжруулалт

1. **Ээлж дуусгах удирдамж** — гарт буй картнуудын доор `[Ээлж дуусгах: -1 оруул]` гэж үргэлж харуулав
2. **BuffCard feedback** — buff идэвхтэй үед "⚔️ Дараагийн attack +2" гэж player line-д нэмэв
3. **Error message уншлага** — 2 секунд зогсоох `Enter дараад үргэлжлүүлнэ үү` үг нэмэв

## Rebuild-ын дараа

(3 сайжруулалтын дараа 1 хүнээр дахин тестлэвэл сайн — гэхдээ хугацаа хүрэлцэхгүй бол заавал биш.)
```

### Үнэлгээ

- 2/5 — Зөвхөн 1 хүн / жагсаалт
- 4/5 — 3 хүн + findings + 1 сайжруулалт
- 5/5 — 3 хүн + findings + **3 хэрэгжсэн сайжруулалт**
